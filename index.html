¿Qué es djangoSIGE?

djangoSIGE es un sistema de gestión empresarial integrado desarrollado en Python y Django. Cubre de forma unificada los procesos centrales del negocio con integridad transaccional garantizada a nivel de base de datos.

Cobertura funcional

Compras, ventas, inventario, finanzas, facturación fiscal y administración de usuarios en un solo sistema.

Tecnologías

Python Django Web Framework Django ORM unittest / TestCase SQLite

Python — Lenguaje de programación principal del sistema, elegido por su claridad sintáctica, robustez y amplio ecosistema de librerías para desarrollo empresarial.

Django Web Framework — Framework de alto nivel que provee estructura MVT (Modelo-Vista-Template), sistema de rutas, capa de abstracción de base de datos y componentes de seguridad integrados.

Django ORM (Object-Relational Mapper) — Capa de persistencia que traduce operaciones Python a SQL, garantizando integridad referencial y permitiendo manipular datos sin escribir consultas SQL explícitas.

unittest / TestCase — Framework nativo de pruebas unitarias, extendido por Django para validar modelos, vistas, formularios y lógica de negocio mediante aserciones y transacciones aisladas.

SQLite — Motor de base de datos relacional utilizado en entornos de desarrollo y pruebas, liviano, sin configuración adicional y con soporte ACID para garantizar consistencia transaccional.

Alcance del sistema

Operaciones CRUD mediante el ORM de Django
Métodos personalizados sobreescritos en modelos
Integridad de datos en operaciones transaccionales
Movimientos de inventario y actualización de existencias
Persistencia y sincronización de datos

Estrategia de verificación

Partición de equivalencia

Cada método se valida con datos representativos válidos e inválidos para cubrir distintas clases de comportamiento.

Valores límite

Se verifican condiciones de borde: presupuestos vencidos, stock en cero, fechas del día, cantidades máximas.

Validación HTTP

Todas las vistas responden con los códigos correctos (200, 404, 500) y renderizan los templates correspondientes.

Validación de formularios

Los campos obligatorios generan FormError al omitirse; ningún dato inválido se persiste en la base de datos.

Verificación de permisos

Usuarios sin los permisos requeridos son redirigidos o bloqueados automáticamente por el sistema.

Pruebas automatizadas

Todos los casos se ejecutan garantizando resultados consistentes entre iteraciones.
