01

¿Qué es djangoSIGE?

djangoSIGE es un Sistema Integrado de Gestión Empresarial (ERP) de código abierto desarrollado en Python 3 con el framework Django 5.2. Cubre de forma unificada los procesos centrales del negocio con integridad transaccional garantizada a nivel de base de datos.

Cobertura funcional

Compras, ventas, inventario, finanzas, facturación fiscal, administración de usuarios, cadastro y más.

Arquitectura

Desplegado con Docker (Nginx + Gunicorn + PostgreSQL 18), lo que garantiza un entorno aislado y reproducible.

Tecnologías del proyecto

Python Django Web Framework Django ORM unittest / TestCase SQLite
Python
Lenguaje principal del sistema. Elegido por su claridad sintáctica, robustez y amplio ecosistema para desarrollo empresarial.
Django
Framework de alto nivel con estructura MVT, sistema de rutas, abstracción de base de datos y componentes de seguridad integrados.
Django ORM
Capa de persistencia que traduce operaciones Python a SQL, garantizando integridad referencial sin consultas SQL explícitas.
unittest
Framework nativo extendido por Django para validar modelos, vistas, formularios y lógica de negocio con transacciones aisladas.
SQLite
Motor relacional liviano para entornos de prueba, con soporte ACID para garantizar consistencia transaccional completa.

08

Estrategia de verificación

Partición de equivalencia

Cada método se valida con datos representativos válidos e inválidos para cubrir distintas clases de comportamiento.

Valores límite

Se verifican condiciones de borde: presupuestos vencidos, stock en cero, fechas del día, cantidades máximas.

Validación HTTP

Todas las vistas responden con los códigos correctos (200, 404, 500) y renderizan los templates correspondientes.

Validación de formularios

Los campos obligatorios generan FormError al omitirse; ningún dato inválido se persiste en la base de datos.

Verificación de permisos

Usuarios sin los permisos requeridos son redirigidos o bloqueados automáticamente por el sistema.

Pruebas automatizadas

Todos los casos se ejecutan garantizando resultados consistentes entre iteraciones de prueba.

Transición de estados

Se modelan y evalúan las transiciones legales e ilegales de documentos (presupuesto → pedido → cancelado/recibido).

Tablas de decisión

Combinación de condiciones complejas para validar reglas de negocio (ej. cálculo de impuestos, descuentos).

Pipeline CI/CD con GitHub Actions

El pipeline de integración continua se define en el archivo .github/workflows/unit_tests.yml y automatiza la ejecución de las pruebas unitarias ante cada evento de push o pull_request dirigido a las ramas main y develop. Su propósito es garantizar que cualquier cambio introducido al código sea validado de forma automática e inmediata, sin depender de la ejecución manual por parte del equipo.

Paso 1 Checkout del repositorio actions/checkout@v4 Descarga el código fuente actualizado en el runner de GitHub.
Paso 2 Construcción de imagen Docker docker build -t djangosige . Crea una imagen contenedorizada del proyecto, asegurando que las pruebas corran sobre el mismo entorno definido para QA.
Paso 3 Ejecución de pruebas unitarias python manage.py test (6 módulos) Ejecuta los 245 casos de prueba sobre compras, estoque y vendas (models + forms).

GitHub Actions Archivo de configuración: .github/workflows/unit_tests.yml · Módulos probados: compras (models + forms), estoque (models + forms), vendas (models + forms) · El pipeline actúa como puerta de calidad obligatoria: ningún cambio puede ser integrado a las ramas protegidas sin haber superado satisfactoriamente todos los casos de prueba.

Beneficios del pipeline: • Validación automática en cada commit • Entorno Docker consistente • Resultados visibles en cada PR • Reducción de errores en producción
